# Инструкция по запуску RAG-системы

## Описание

RAG-система для ответов на вопросы по **Национальной стратегии развития искусственного интеллекта на период до 2030 года**.

Пользователь вводит вопрос — система находит релевантные фрагменты документа и генерирует ответ через LLM (Ollama).

---

## Структура проекта

```
rag_dz/
├── engine.py        — построение FAISS индекса из PDF
├── app.py           — веб-интерфейс (Streamlit)
├── testing.py       — оценка качества (RAGAS)
├── requirements.txt — зависимости Python
├── data/
│   ├── data_text.pdf       — исходный документ
│   ├── faiss_index.bin     — FAISS индекс (уже готов)
│   ├── chunks.pkl          — текстовые чанки (уже готов)
│   ├── test_set_v2.xlsx    — тестовый набор (question, ground_truth, answer)
│   └── results.json        — результаты последнего RAGAS-тестирования
└── venv_faiss/      — виртуальное окружение Python
```

## Шаг 1 — Установка зависимостей

### 1.1 Создать виртуальное окружение

```bash
python3.10 -m venv venv_faiss
source venv_faiss/bin/activate
```

### 1.2 Установить пакеты

```bash
pip install -r requirements.txt -i https://pypi.org/simple/
```

> Установка займёт 5–10 минут (скачивается PyTorch, sentence-transformers и др.)

---

## Шаг 2 — Установка и запуск Ollama

### 2.1 Установить Ollama

```bash
# macOS
brew install ollama

# Linux
curl -fsSL https://ollama.com/install.sh | sh
```

### 2.2 Запустить сервер Ollama

**Вариант A — в фоне (рекомендуется, терминал не блокируется):**

```bash
ollama serve > /tmp/ollama.log 2>&1 &
```

Проверить что запустился:

```bash
curl -s http://localhost:11434 && echo "Ollama работает"
```

Остановить при необходимости:

```bash
pkill ollama
```

**Вариант B — в отдельном терминале:**

```bash
ollama serve
```

> Оставить окно терминала открытым — закрытие остановит сервер.

### 2.3 Скачать модель

```bash
ollama pull qwen2.5:7b
```

> Скачивание займёт 5–15 минут (~4.7 ГБ). Модель оптимизирована для русского языка.

## Шаг 3 — Построение FAISS индекса (опционально)

Выполняется только при смене PDF-документа. **Индекс уже готов — этот шаг можно пропустить.**

```bash
source venv_faiss/bin/activate
python3.10 engine.py
```

**Что происходит:**
1. Читается `data/data_text.pdf`
2. Текст разбивается на чанки (гибридный метод: по пунктам + razdel)
3. Чанки кодируются моделью `multilingual-e5-base`
4. Создаётся `data/faiss_index.bin` и `data/chunks.pkl`

**Ожидаемый вывод:**
```
Чтение PDF: data/data_text.pdf
Извлечено символов: 76993
Чанков: 197
Загрузка модели intfloat/multilingual-e5-base...
Кодирование чанков...
FAISS индекс построен: 197 векторов, размерность 768
```

> Время: 1–3 минуты (зависит от ЦП/GPU)

## Шаг 4 — Запуск веб-приложения

```bash
source venv_faiss/bin/activate
streamlit run app.py
```

Приложение откроется в браузере по адресу: **http://localhost:8501**

**Использование:**
1. Введите вопрос в поле «Вопрос»
2. Нажмите «Получить ответ»
3. Система найдёт релевантные фрагменты и сгенерирует ответ через Ollama
4. Ответ отобразится в поле ниже

> Если Ollama недоступна — отображаются релевантные фрагменты из документа напрямую

---

## Шаг 5 — Запуск RAGAS-тестирования (опционально)

Оценивает качество RAG по 5 метрикам на тестовом наборе из `data/test_set_v2.xlsx`.

```bash
source venv_faiss/bin/activate
python3.10 testing.py
```

**Метрики:**
- **Answer Relevancy** — насколько ответ по существу вопроса
- **Answer Correctness** — фактическая корректность относительно ground_truth
- **Answer Similarity** — близость к эталону (семантическое сходство)
- **Clarity** (AspectCritic) — понятность и логичность формулировки
- **Safety** (AspectCritic) — отсутствие вредного/неэтичного содержания

**Тестовый набор (`data/test_set_v2.xlsx`):**
- Колонки: `question`, `ground_truth`, `answer`
- Если `ground_truth` пуст — автоматически заполняется ответом модели при запуске скрипта

**Достигнутые результаты:**

| Метрика | Значение |
|---------|----------|
| Answer Relevancy | 0.5319 |
| Answer Correctness | 0.5785 |
| Answer Similarity | 0.8787 |
| Clarity | 1.0000 |
| Safety | 1.0000 |

Детальные результаты сохраняются в `data/results.json` (итоговые оценки + разбивка по каждому вопросу).

> Время выполнения: 15–25 минут

## Технические детали

| Параметр | Значение |
|----------|----------|
| LLM | `qwen2.5:7b` (Ollama) |
| Temperature | `0` (детерминированные ответы) |
| Embeddings | `intfloat/multilingual-e5-base` |
| Vector store | FAISS (IndexFlatIP, cosine similarity) |
| Чанкинг | Гибридный: по пунктам → razdel → посимвольно |
| Размер чанка | 600 символов, перекрытие 120 |
| top_k | 5 чанков при поиске |

## Возможные проблемы

| Проблема | Решение |
|----------|---------|
| `could not connect to ollama server` | Запустить `ollama serve` в отдельном терминале |
| `data/faiss_index.bin not found` | Выполнить `python3.10 engine.py` для построения индекса |
| LLM не отвечает / пустой ответ | Проверить что модель скачана: `ollama list` |
| Медленная работа | Нормально для CPU: qwen2.5:7b генерирует ~10–15 токенов/сек на M1/M2 |
| TimeoutError в RAGAS | Увеличить `timeout` в `RunConfig` в `testing.py` |
