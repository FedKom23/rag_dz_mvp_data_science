# Инструкция по запуску RAG-системы

## Описание

RAG-система для ответов на вопросы по **Национальной стратегии развития искусственного интеллекта на период до 2030 года**.

Пользователь вводит вопрос — система находит релевантные фрагменты документа и генерирует ответ через LLM (Ollama).

---

## Структура проекта

```
rag_dz/
├── engine.py        — построение FAISS индекса из PDF
├── app.py           — веб-интерфейс (Streamlit)
├── testing.py       — оценка качества (RAGAS)
├── requirements.txt — зависимости Python
├── data/
│   ├── data_text.pdf       — исходный документ
│   ├── faiss_index.bin     — FAISS индекс (создаётся автоматически)
│   └── chunks.pkl          — текстовые чанки (создаётся автоматически)
└── venv_faiss/      — виртуальное окружение Python
```

## Шаг 1 — Установка зависимостей

### 1.1 Создать виртуальное окружение

```bash
python3.10 -m venv venv_faiss
source venv_faiss/bin/activate
```

### 1.2 Установить пакеты

```bash
pip install -r requirements.txt -i https://pypi.org/simple/
```

> Установка займёт 5–10 минут (скачивается PyTorch, sentence-transformers и др.)

---

## Шаг 2 — Установка и запуск Ollama

### 2.1 Установить Ollama

```bash
# macOS
brew install ollama

# Linux
curl -fsSL https://ollama.com/install.sh | sh
```

### 2.2 Запустить сервер Ollama

```bash
ollama serve
```

> Оставить запущенным в отдельном терминале (или в фоне: `ollama serve &`)

### 2.3 Скачать модель

```bash
ollama pull llama3.1:8b
```

> Скачивание займёт 5–15 минут (~4.9 ГБ)

## Шаг 3 — Построение FAISS индекса

Выполняется **один раз** при первом запуске или при смене PDF-документа.

```bash
source venv_faiss/bin/activate
python3.10 engine.py
```

**Что происходит:**
1. Читается `data/data_text.pdf`
2. Текст разбивается на чанки (гибридный метод: по пунктам + razdel)
3. Чанки кодируются моделью `multilingual-e5-base`
4. Создаётся `data/faiss_index.bin` и `data/chunks.pkl`

**Ожидаемый вывод:**
```
Чтение PDF: data/data_text.pdf
Извлечено символов: 76993
Чанков: 197
Загрузка модели intfloat/multilingual-e5-base...
Кодирование чанков...
FAISS индекс построен: 197 векторов, размерность 768
```

> Время: 1–3 минуты (зависит от ЦП/GPU)

## Шаг 4 — Запуск веб-приложения

```bash
source venv_faiss/bin/activate
streamlit run app.py
```

Приложение откроется в браузере по адресу: **http://localhost:8501**

**Использование:**
1. Введите вопрос в поле «Вопрос»
2. Нажмите «Получить ответ»
3. Система найдёт релевантные фрагменты и сгенерирует ответ через Ollama
4. Ответ отобразится в поле ниже

> Если Ollama недоступна — отображаются релевантные фрагменты из документа напрямую

---

## Шаг 5 — Запуск RAGAS-тестирования (опционально)

Оценивает качество RAG по 4 метрикам на тестовом наборе.

```bash
source venv_faiss/bin/activate
python3.10 testing.py
```

**Метрики:**
- **Faithfulness** — ответ подкреплён контекстом (нет галлюцинаций)
- **Answer Relevancy** — ответ релевантен вопросу
- **Context Precision** — найденные чанки релевантны вопросу
- **Context Recall** — найденные чанки покрывают нужную информацию

**Ожидаемые результаты (llama3.1:8b, top_k=5):**
```
Faithfulness       (достоверность): ~0.94
Answer Relevancy   (релевантность): ~0.92
Context Precision  (точность):      ~0.91
Context Recall     (полнота):       ~0.67
```

Детальные результаты сохраняются в `data/ragas_results.csv`.

> Время выполнения: 10–15 минут (модель обрабатывает 20 LLM-запросов)

## Возможные проблемы

| Проблема | Решение |
|----------|---------|
| `could not connect to ollama server` | Запустить `ollama serve` в отдельном терминале |
| `data/faiss_index.bin not found` | Выполнить `python3.10 engine.py` для построения индекса |
| LLM не отвечает / пустой ответ | Проверить что модель скачана: `ollama list` |
| Медленная работа | Нормально для CPU: llama3.1:8b генерирует ~10–15 токенов/сек на M1/M2 |
| TimeoutError в RAGAS | Увеличить `timeout` в `RunConfig` в `testing.py` |
